## Data Flow

1. **User Request** вҶ’ Internet вҶ’ AWS WAF (security filtering)
2. **WAF** вҶ’ Application Load Balancer (SSL termination, routing)
3. **ALB** вҶ’ EKS Service вҶ’ Pods (across multiple AZs)
4. **Pods** вҶ’ RDS PostgreSQL (persistent data)
5. **Pods** вҶ’ ElastiCache Redis (caching, sessions)
6. **Pods** вҶ’ CloudWatch/Prometheus (metrics, logs)

## Auto-Scaling Flow

### Normal Traffic (Baseline)
- 3 pods across 3 nodes (1 per AZ)
- CPU: 20-30%, Memory: 40-50%

### Flash Sale (10x Traffic Spike)
1. **T+0s**: Traffic spike detected
2. **T+15s**: HPA scales pods from 3 вҶ’ 30 (100% increase)
3. **T+30s**: Cluster Autoscaler adds nodes
4. **T+60s**: New nodes ready, pods scheduled
5. **T+90s**: System stabilized at 30-50 pods

### Post-Flash Sale (Scale Down)
1. **T+5min**: Traffic normalizes
2. **T+10min**: HPA gradually reduces pods (10% per minute)
3. **T+5min**: Karpenter consolidates and removes unused nodes
4. **T+10min**: System back to baseline (3 pods)

## Failure Scenarios & Recovery

### Scenario 1: AZ Failure
- **Detection**: 30 seconds (health checks)
- **Action**: ALB stops routing to failed AZ
- **Recovery**: Pods rescheduled to healthy AZs
- **Impact**: Zero downtime (PDB ensures min 2 pods)

### Scenario 2: Bad Deployment
- **Detection**: Readiness probe failures
- **Action**: Rolling update halts (maxUnavailable=0)
- **Recovery**: Automatic rollback via kubectl
- **Impact**: No user impact (old pods still serving)

### Scenario 3: Database Failover
- **Detection**: RDS automatic (60 seconds)
- **Action**: DNS updated to standby
- **Recovery**: Application reconnects automatically
- **Impact**: 60-120 seconds of degraded performance

### Scenario 4: Node Failure
- **Detection**: 40 seconds (kubelet heartbeat)
- **Action**: Pods rescheduled to healthy nodes
- **Recovery**: Automatic via Kubernetes
- **Impact**: Minimal (anti-affinity ensures distribution)

## Security Layers

1. **Perimeter**: WAF with rate limiting and OWASP rules
2. **Network**: Security Groups, Network Policies
3. **Application**: Pod Security Context, RBAC
4. **Data**: Encryption at rest (KMS), in transit (TLS)
5. **Secrets**: AWS Secrets Manager, IRSA
6. **Audit**: CloudWatch Logs, VPC Flow Logs

## Cost Optimization Strategies

1. **Spot Instances**: Use for burst capacity (c5.2xlarge)
2. **Right-sizing**: Resource requests match actual usage
3. **Auto-scaling**: Scale down during low traffic
4. **Reserved Instances**: For baseline capacity (future)
5. **S3 Lifecycle**: Archive old logs to Glacier
